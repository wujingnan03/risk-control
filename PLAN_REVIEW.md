# 基金交易风控 Skill 方案审查与修订

## 一、原方案问题诊断

### 🔴 严重问题

**1. "基于已标记样本训练" 概念错位**
- Skill 不是 ML 模型，不能"训练"。正确做法是：Claude 分析已标记 UID 的行为数据，**归纳提炼特征模式和规则**，写入反欺诈知识库
- 修正方案：改为"特征提炼模式"——输入标记样本 → 调 MCP 拉取数据 → Python 统计分析 → 输出规则/特征模式 → 追加到 references/anti_fraud_rules.md

**2. "业务知识自行到网上学习" 不可控**
- 运行时联网学习会导致：每次结果不同、延迟高、可能学到错误信息
- 修正方案：**预置知识库**（references/ 目录），包含雪球基金业务说明、反洗钱法核心条文、反欺诈规则库。运行时只查引用，不联网

**3. 稳定性缺乏工程保障**
- 仅靠"要求不空不乱填"不够，需要：
  - 严格的 JSON Output Schema（每个模块的输出格式固定）
  - Python 脚本做计算（确定性逻辑不靠 LLM 生成）
  - 输出校验脚本（检查必填字段、值域、类型）

### 🟡 中等问题

**4. 模块间数据流未定义**
- 四个模块串行依赖，但没有定义：每个模块的输入是什么？输出是什么？上游缺失数据时怎么处理？
- 修正：定义明确的模块接口契约（Module Interface Contract）

**5. MCP 调用缺少容错策略**
- MCP 接口可能超时、返空、返错。未定义：重试策略？降级方案？数据缺失时如何标注？
- 修正：每次 MCP 调用后做数据完整性检查，缺失字段标注为 `"数据缺失"` 而非留空或编造

**6. 人群模块两个功能耦合**
- (1) 特征提炼 和 (2) 可疑用户识别 是完全不同的工作流，不应耦合在一个模块
- 修正：拆为两个独立工作流（workflow），共享同一套 MCP 和 Python 工具

### 🟢 建议优化

**7. 迁移性设计**
- 要迁移到其他 agent / OpenClaw，模块应设计为"可独立调用"
- 每个模块的 references 和 scripts 自包含，不依赖特定平台 API

**8. 调试支持**
- 增加 `--debug` 模式：输出中间数据（MCP 原始返回、Python 计算中间结果）
- 每个模块输出附带 `_debug` 字段

## 二、修订后的架构设计

```
fund-risk-control/
├── SKILL.md                          # 主调度：工作流编排 + 模块路由
├── references/
│   ├── business_context.md           # 雪球基金业务说明（预置）
│   ├── anti_fraud_rules.md           # 反欺诈规则库（预置 + 可追加）
│   ├── anti_money_laundering.md      # 反洗钱规则库（基于法规，预置）
│   └── output_schema.md             # 各模块输出 JSON Schema
├── scripts/
│   ├── user_profiling.py             # 用户行为画像计算（确定性逻辑）
│   ├── risk_scoring.py               # 风险评分模型
│   └── validate_output.py            # 输出校验
└── evals/
    └── evals.json                    # 测试用例
```

### 工作流 A：单用户风控评估（核心流程）
```
输入 UID → MCP拉数据 → 行为画像(Python) → 反欺诈分析 → 反洗钱分析 → 综合评分
```

### 工作流 B：特征提炼（离线分析）
```
输入标记UID列表 → 批量MCP拉数据 → Python统计 → 归纳特征模式 → 写入规则库
```

## 三、各模块接口契约

| 模块 | 输入 | 输出 | 确定性保障 |
|------|------|------|-----------|
| 数据采集层 | UID | 用户信息 + 设备 + 登录日志 + IP | MCP 调用 + 完整性校验 |
| 行为画像 | 原始数据 JSON | 画像 JSON（维度固定） | Python 脚本计算 |
| 反欺诈 | 画像 + 规则库 | 命中规则列表 + 风险标签 | 规则匹配（LLM 做模糊推理） |
| 反洗钱 | 画像 + 设备 + 交易 | 命中规则列表 + 风险标签 | 规则匹配（LLM 做模糊推理） |
| 综合评分 | 所有模块输出 | 评分 + 级别 + 建议 + 证据链 | Python 评分 + LLM 总结 |
