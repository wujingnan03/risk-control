## 规则详情

### AML-001 单日大额交易报告
- **条件**：单日（按 tradeDate 分组）累计买入金额（amount字段，tradeType=买入/定投） ≥ 人民币 5 万元
- **严重程度**：高
- **法规依据**：根据《金融机构大额交易和可疑交易报告管理办法》，基金销售机构应对大额交易进行监控报告
- **证据要求**：tradeDate、当日所有交易明细（tradeType + fundName + fundCode + amount/shares）、日累计amount
- **数据字段**：依赖 amount 字段（仅买入/定投有值）。卖出交易只有 shares 无 amount，无法直接计算卖出金额
- **注意**：大额交易不等于洗钱，但需关注是否伴随其他异常

### AML-002 资金快进快出
- **条件**：同一 fundCode 上，买入（tradeType=买入/定投）后 7 天内出现卖出（tradeType=卖出），且买入 amount ≥ 1 万元
- **严重程度**：高
- **风险说明**：资金短期内快速流转是典型洗钱特征之一，通过基金申赎实现资金"清洗"
- **证据要求**：买入记录（tradeDate + fundCode + fundName + amount）、卖出记录（tradeDate + fundCode + shares）、持有天数
- **交叉验证**：结合 fundCode 判断基金类型。货币基金短期申赎较正常；权益基金短期大额赎回更可疑
- **数据限制**：卖出只有 shares 无 amount，无法直接比对金额，以买入 amount 为判断依据

### AML-003 交易拆分规避报告
- **条件**：单日（tradeDate）内 tradeType=买入/定投 的交易 ≥ 3 笔，且单笔 amount 在 4-5 万元之间（疑似规避 5 万元大额报告门槛）
- **严重程度**：高
- **风险说明**：刻意将大额交易拆分为多笔小额交易以规避大额交易报告义务
- **证据要求**：当日所有买入交易（tradeDate + fundCode + fundName + amount）、金额分布、是否接近报告阈值
- **交叉验证**：分析该模式是否持续出现（多个 tradeDate 命中）。偶发一次可能是正常操作，连续多日则高度可疑
- **脚本支持**：user_profiling.py 已计算 `split_buy_days` 字段

### AML-004 频繁申赎同一产品
- **条件**：30 天内对同一 fundCode 的买入（tradeType=买入/定投）和卖出（tradeType=卖出）各 ≥ 3 次
- **严重程度**：中
- **风险说明**：反复申赎同一产品且无明显盈利目的，可能是利用基金通道转移资金
- **证据要求**：fundCode + fundName、买入次数及明细、卖出次数及明细、时间跨度
- **交叉验证**：如果频繁操作但无明显收益目的，更加可疑

### AML-005 短期集中大额赎回
- **条件**：7 天内卖出（tradeType=卖出）交易笔数 ≥ 5 笔，涉及多个 fundCode
- **严重程度**：高
- **风险说明**：短期内大量赎回可能是转移非法所得或逃避监管
- **证据要求**：各笔卖出记录（tradeDate + fundCode + fundName + shares）
- **数据限制**：卖出交易只有 shares 无 amount，无法直接计算赎回总金额。以卖出笔数和涉及基金数量作为替代判断维度
- **交叉验证**：结合市场环境。如果大盘暴跌期间赎回，可能是正常止损

### AML-006 客户身份信息异常 [已禁用]
- **条件**：以下任一满足：
  - 注册信息中关键字段（姓名、证件号、手机号）与公安系统不匹配
  - 证件即将过期或已过期未更新
  - 预留手机号非实名
- **严重程度**：中
- **状态**：已禁用
- **禁用原因**：当前无外部身份核验系统对接，MCP 不提供公安系统比对、证件有效期、手机实名状态等数据
- **法规依据**：《反洗钱法》客户身份识别义务
- **证据要求**：异常字段、具体不匹配内容
- **重新启用条件**：接入身份核验（公安二要素/运营商三要素）接口后可重新启用

### AML-007 代理人/关联人代操作 [已禁用]
- **条件**：以下任一满足：
  - 同一设备登录 ≥ 3 个不同账号
  - 同一 IP 在同一时间段操作多个账号
  - 用户信息中的联系方式与其他账号重复
- **严重程度**：高
- **状态**：已禁用
- **禁用原因**：需要跨账户关联查询能力，当前 MCP 仅支持单 UID 查询，无法获取同设备/IP 下的其他账号信息
- **风险说明**：可能存在他人代为操控账号，规避实际受益人识别
- **证据要求**：共享设备/IP 的账号列表、重叠时间段
- **交叉验证**：结合账号之间是否有资金往来关系
- **重新启用条件**：MCP 提供跨账户设备/IP 关联查询接口后可重新启用

### AML-008 资金来源与客户身份不匹配 [已禁用]
- **条件**：交易金额与客户的收入水平、职业、年龄明显不匹配
- **严重程度**：高
- **状态**：已禁用
- **禁用原因**：当前无收入水平、职业等外部数据源，MCP 不提供客户经济能力评估所需字段，无法进行确定性匹配
- **风险说明**：超出客户合理经济能力的交易可能涉及非法资金
- **证据要求**：客户基本信息（年龄、职业）、交易金额级别
- **交叉验证**：需结合用户风险评级和适当性评估结果
- **重新启用条件**：接入客户收入/职业数据源后可重新启用

### AML-009 多账户同设备/IP
- **条件**：同一设备 ID 或 IP 地址在 24 小时内登录 ≥ 2 个不同用户账号
- **严重程度**：高
- **风险说明**：可能是同一人操控多个"空壳账户"实施洗钱
- **证据要求**：设备 ID/IP、关联的 UID 列表、登录时间
- **注意**：本规则的完整验证需要跨账户数据，当前 MCP 接口可能仅支持单 UID 查询。如果数据不足，标注为"数据受限，无法完整验证"

### AML-010 交易操作与登录地异常
- **条件**：发生交易时的登录 IP 与用户常驻地（近 90 天最常出现的省份）不一致
- **严重程度**：中
- **风险说明**：异地操作大额交易需关注
- **证据要求**：交易时 IP 和归属地、用户常驻地
- **交叉验证**：结合是否为已知出差或旅行时段（通常无法判断，标注为待确认）

### AML-011 无合理经济目的交易 [已禁用]
- **条件**：交易模式不符合正常投资逻辑，如：
  - 在赎回确认到账前又发起同金额申购
  - 频繁在不同基金间转换但无收益动机
  - 高频小额交易远超正常投资频率
- **严重程度**：高
- **状态**：已禁用
- **禁用原因**：纯定性判断，无可量化的确定性条件，依赖主观推理，不符合"数值计算由 Python 脚本执行"的确定性原则
- **法规依据**：《反洗钱法》第十九条关于可疑交易的定义
- **证据要求**：交易流水摘要、异常模式描述
- **重新启用条件**：提炼出可量化的子条件（如"同金额申赎间隔<T+2"）后可重新启用

### AML-012 多项洗钱指标叠加
- **条件**：AML-001 至 AML-011 中**已启用**规则同时命中 ≥ 3 条
- **严重程度**：高
- **风险说明**：多项指标同时触发表明洗钱风险极高
- **证据要求**：列出所有命中规则及各自证据

## 处置建议映射

| 命中情况 | 建议处置 |
|---------|---------|
| 仅 AML-001（大额交易）且无其他异常 | 正常大额交易报告，持续监控 |
| 命中 1 条中等严重度规则 | 列入关注名单，加强后续交易监控 |
| 命中 1 条高严重度规则 | 限制大额交易，人工复核，考虑提交可疑交易报告 |
| 命中 ≥ 2 条高严重度规则 | 暂停交易权限，立即人工审查，提交可疑交易报告 |
| 命中 AML-012（多指标叠加） | 冻结账户操作，提交可疑交易报告，报告合规部门 |

## 规则维护说明

- 新规则通过工作流 B（特征提炼）生成建议，经合规部门审核后添加
- 规则 ID 格式：`AML-{三位数字}`，从 013 起递增
- 法规更新时需同步更新对应规则的 `regulatory_reference`
- 阈值调整需合规部门批准并记录变更原因
